# 第 11 章两条大模型诊断案例实验说明

## 实验名称

第 11 章两条大模型诊断案例实验说明

## 实验目标

- 帮助读者把教材第 11 章中的“大模型基本原理、监督微调与 RAG”落到两个具体诊断案例上。
- 对比“特征文本化 + ChatGLM2-6B-chat”和“振动序列补丁化 + GPT-2”两条路线的输入形式、建模思路和适用边界。
- 为后续课程设计或实验教学提供可直接讲解的实验说明模板，而不是只留下文献摘要式描述。

## 实验知识点

- 2048 点振动窗口切片
- 24 个时域与频域统计特征
- 监督微调样本组织
- LoRA 与 QLoRA 的使用逻辑
- patching、一维卷积嵌入与位置编码
- 多数据集、跨工况与小样本迁移的结果解读

## 实验环境与数据准备

- Python 3.10 或以上
- 建议安装 `numpy`、`pandas`、`scipy`、`torch`、`transformers`、`peft` 和 `datasets`
- 若只做教学讲解，不强制要求完整训练环境；若希望复现参考案例流程，建议使用具有 24 GB 左右显存的 GPU 环境
- 数据准备以 CWRU、MFPT、JNU 和 PU 四个数据集为主，样本窗口长度默认采用 2048

## 目录说明

- `exp01_feature_textualized_chatglm.md`：案例一实验说明，聚焦“24 个统计特征文本化 + ChatGLM2-6B-chat”
- `exp02_patched_gpt2.md`：案例二实验说明，聚焦“原始振动序列 patching + GPT-2”

## 建议使用顺序

1. 先阅读 `exp01_feature_textualized_chatglm.md`，理解大模型如何接入前面章节的特征工程结果。
2. 再阅读 `exp02_patched_gpt2.md`，理解大模型如何通过输入适配承担时序分类任务。

## 补充说明

本目录的目标是提供“教材可讲、实验可解释、结构可复用”的说明文档，不承诺完整训练工程。若后续确需落成可运行代码，应在此基础上进一步补全数据下载、特征提取、训练脚本和日志记录模块。

## 实验总结

这两个案例共同说明，大模型进入智能运维并不只有一种方式。若读者已经掌握前面章节中的特征工程和监督分类，就可以先从特征文本化路线理解大模型的语言接口；若希望进一步理解大模型如何变成时序建模骨干，则应继续学习序列补丁化路线。两条路线一起看，才能真正看出第 11 章的亮点所在。
