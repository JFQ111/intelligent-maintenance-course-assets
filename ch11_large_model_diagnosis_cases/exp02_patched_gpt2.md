# 振动序列补丁化 + GPT-2 实验说明

## 实验名称

第 11 章 案例二：振动序列补丁化 + GPT-2

## 实验目标

- 理解为什么原始振动序列不能直接等同于语言标记输入，而需要 patching 和嵌入适配。
- 掌握“patching -> 一维卷积嵌入 -> 位置编码 -> GPT-2 主干 -> 分类头”的完整建模链条。
- 理解冻结多头自注意力层和前馈网络层、仅训练 LayerNorm 与位置嵌入的设计动机。
- 学会结合单数据集、跨工况和跨数据集结果评价这一路线的优缺点。

## 实验知识点

- 2048 点窗口样本
- patch size 与 stride 的作用
- 一维卷积嵌入
- 正弦位置编码
- GPT-2 主干适配
- 实例归一化与仿射变换
- 交叉熵损失

## 实验环境与数据准备

建议环境如下：

- Python 3.10 或以上
- `numpy`、`pandas`、`scipy`
- `torch`
- `transformers`
- 建议准备 GPU 环境以支持时序模型训练

数据准备与案例一相同，仍然使用 CWRU、MFPT、JNU 和 PU 四个数据集。所有原始振动序列先切成 2048 点窗口样本。与案例一不同，本案例不再提取 24 个统计特征，而是把每个 2048 点窗口直接送入 patching 模块。

## 实验原理

本案例的出发点，是尽量保留原始振动序列中的局部结构，不让信息在特征提取阶段被过度压缩。但 GPT-2 的原始输入接口面向的是文本标记序列，而不是高频连续采样点。因此，模型不能直接读取 2048 个实数，而必须先把时序数据改写为与语言模型输入维度兼容的表示。

参考案例采用的第一步是 patching。它把长度为 2048 的序列切成多个重叠 patch，每个 patch 再通过一维卷积映射到模型输入维度。若 patch size = 128、stride = 8，则一个 2048 点窗口会被切成 241 个 patch。这样做的好处是，模型处理的是“局部片段序列”，而不是逐点原始序列，计算负担显著下降。

第二步是位置编码。由于 patch 序列本身不携带顺序信息，需要额外加入正弦位置编码，让模型知道每个 patch 在原始时间序列中的相对位置。第三步是主干适配。参考案例冻结了 GPT-2 的多头自注意力层和前馈网络层，只训练 LayerNorm 和位置嵌入相关参数，以减少训练成本并保留预训练先验。

第四步是归一化与损失函数。作者先对输入振动序列做实例归一化，再在 LayerNorm 中保留可学习仿射变换参数。这种做法有助于缓解不同数据集和不同工况下的幅值分布差异。损失函数采用交叉熵损失，说明该案例的最终目标仍是监督分类，而不是开放式文本生成。

## 实验内容与步骤

### 步骤 1 窗口切片与数据划分

先把四个数据集中的原始振动信号切成 2048 点窗口样本。然后按 8:1:1 划分训练集、验证集和测试集，并尽量保持各故障类别样本数相近。教学时应强调，验证集的作用是帮助选择最佳 epoch，而不是可有可无的附加步骤。

### 步骤 2 patching 与嵌入

以 patch size = 128、stride = 8 为例，把每个 2048 点窗口切成 241 个 patch。然后通过一维卷积层把每个 patch 映射到 GPT-2 的输入维度。这里的一维卷积层并不是普通分类网络的第一层卷积，而是承担“时间序列版 token embedding”的角色。

### 步骤 3 位置编码与主干适配

为每个 patch 叠加正弦位置编码，再将 patch 序列送入 GPT-2 主干。主干中冻结多头自注意力层和前馈网络层，只训练 LayerNorm 与位置嵌入相关参数。教学时可以把这一策略解释为“保留大模型已有的序列关系建模能力，只在最靠近任务接口的位置做轻量适配”。

### 步骤 4 归一化、训练与验证

对输入做实例归一化，并保留 LayerNorm 中的可学习仿射变换参数。根据参考案例的单数据集实验设定，可使用 learning rate = 0.001、epoch = 50，并在每轮训练后用验证集选择最佳模型。超参数选择实验表明，更大的 patch 和更小的 stride 往往有助于提高准确率，但会增加训练时间，因此最终配置体现的是性能与成本的折中。

### 步骤 5 结果分析

参考案例报告的单数据集准确率为：CWRU 约 0.9996、MFPT 约 0.9982、JNU 约 0.9981、PU 约 0.9239。跨数据集完整迁移实验中，PU、MFPT、JNU 和 CWRU 的结果分别约为 0.9338、0.9958、0.9988 和 0.9987。小样本继续微调实验中，PU、MFPT 和 JNU 的准确率分别提升约 0.0396、0.0376 和 0.0327，CWRU 仅提升约 0.0028。教学时应特别指出：这些数字不是为了证明“大模型必胜”，而是为了展示适配式大模型主干在不同数据分布下的收益差异。

## 实验练习题

- 为什么高采样率振动序列不适合逐点直接送入 GPT-2，而更适合先做 patching？
- 在 patch size 与 stride 的选择上，为什么会出现“精度提升但训练时间上升”的权衡？
- 为什么本案例冻结多头自注意力层和前馈网络层，而不是直接做全参数微调？
- 实例归一化和可学习仿射变换为什么对跨数据集适应有帮助？

## 补充说明

本案例比特征文本化路线更接近端到端时序建模，因此工程门槛也更高。若课堂时间有限，可以把它作为“提高型案例”讲解，重点放在输入适配逻辑和实验结果解读上，而不必一开始就要求学生完整实现全部模块。若要扩展为完整代码工程，建议单独增加数据管道、训练日志和超参数搜索脚本。

## 实验总结

本案例说明，大模型不仅可以作为文本问答接口，也可以通过 patching 和嵌入适配成为振动时序分类主干。它比特征文本化路线更少依赖人工特征，更强调原始序列建模，但工程复杂度也更高。将它与案例一并列学习，读者就能更全面地理解“大模型进入智能运维”的两种典型思路。
